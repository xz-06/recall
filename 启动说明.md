# 🚀 启动说明

## 快速启动

### Windows用户

双击运行 `start.bat` 文件即可启动开发服务器。

### 手动启动

```bash
npm run dev
```

## 访问地址

启动成功后，访问：

- **回忆录网站**：http://localhost:8000
- **上传页面**：http://localhost:3001/upload

## 常见问题

### 问题1：mkdocs命令未找到

**错误信息**：
```
'mkdocs' is not recognized as an internal or external command
```

**原因**：MkDocs已安装但不在系统PATH中（使用`--user`安装时常见）

**解决方法**：

1. **使用python -m mkdocs**（已自动配置）
   - 脚本已更新为使用 `python -m mkdocs`
   - 无需额外操作

2. **手动添加到PATH**（可选）
   - 找到Python用户脚本目录：`%APPDATA%\Python\Python39\Scripts`
   - 添加到系统PATH环境变量

3. **重新安装到系统目录**（需要管理员权限）
   ```bash
   pip install -r requirements.txt
   ```

### 问题2：端口被占用

如果8000或3001端口被占用：

1. **修改MkDocs端口**
   - 编辑 `package.json` 中的 `mkdocs:serve` 脚本
   - 修改端口号，例如：`-a 127.0.0.1:8001`

2. **修改后端端口**
   - 编辑 `server/index.js`
   - 修改 `PORT` 变量

### 问题3：站点未构建

**错误信息**：
```
站点未构建
请先运行: npm run mkdocs:build
```

**解决方法**：

```bash
# 构建站点
npm run mkdocs:build

# 然后启动开发服务器
npm run dev
```

### 问题4：中文字符显示问题

如果看到乱码：

1. 确保终端支持UTF-8编码
2. 使用PowerShell而不是CMD
3. 手动设置编码：
   ```bash
   chcp 65001
   ```

## 停止服务器

按 `Ctrl + C` 停止服务器。

## 生产环境启动

```bash
# 构建站点
npm run mkdocs:build

# 启动生产服务器
npm start
```
