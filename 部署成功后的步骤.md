### 步骤2：部署后端API（如果需要上传功能）⚠️

**重要**：Vercel只部署了静态站点，上传功能需要单独的后端服务器。

#### 选项A：使用Railway（推荐）

1. **访问Railway**
   - 打开 https://railway.app
   - 使用GitHub登录

2. **创建新项目**
   - 点击 "New Project"
   - 选择 "Deploy from GitHub repo"
   - 选择仓库：`xz-06/recall`

3. **配置环境变量**
   ```
   PORT = 3001
   NODE_ENV = production
   ```

4. **获取后端地址**
   - Railway会提供一个地址，例如：`https://recall-production.up.railway.app`
   - **复制这个地址！**

5. **更新前端配置**
   
   修改 `docs/js/config.js`（第12行）：
   ```javascript
   const BACKEND_URL = 'https://你的railway地址'; // 替换为实际地址
   ```
   
   修改 `upload.html`（第213行左右）：
   ```javascript
   return 'https://你的railway地址'; // 替换为实际地址
   ```

6. **提交并推送**
   ```bash
   git add docs/js/config.js upload.html
   git commit -m "配置生产环境后端API地址"
   git push origin main
   ```

#### 选项B：使用Render

参考 `部署后端说明.md` 中的Render部署步骤。

### 步骤3：连接GitHub自动部署（可选但推荐）🔗

这样以后推送代码会自动部署：

1. **访问Vercel Dashboard**
   - https://vercel.com/dashboard
   - 选择项目 `recall`

2. **连接GitHub**
   - 进入 **Settings** → **Git**
   - 点击 **Connect Git Repository**
   - 选择 `xz-06/recall`
   - 如果提示需要授权，按照提示操作

3. **完成**
   - 以后推送代码到GitHub，Vercel会自动部署

### 步骤4：配置自定义域名（可选）🌐

如果你想使用自己的域名：

1. **在Vercel Dashboard中**
   - 进入项目 → **Settings** → **Domains**
   - 添加你的域名

2. **配置DNS**
   - 按照Vercel的提示配置DNS记录

3. **等待生效**
   - 通常几分钟内生效

## 🆘 常见问题

### Q: 网站显示404？

A: 
1. 检查 `vercel.json` 配置是否正确
2. 确认 `site/` 目录存在
3. 查看Vercel构建日志

### Q: 上传功能不工作？

A: 
1. 确认后端已部署
2. 检查 `docs/js/config.js` 中的后端地址
3. 检查浏览器控制台错误信息

### Q: 如何更新代码？

A: 
```bash

git push origin main
# Vercel会自动部署
```
