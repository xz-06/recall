# 🚀 快速开始指南

## 第一步：安装依赖

### Windows用户

双击运行 `setup.bat`，或手动执行：

```bash
npm install
pip install -r requirements.txt
```

### Mac/Linux用户

```bash
npm install
pip3 install -r requirements.txt
```

**注意**：确保已安装：
- Node.js（版本14或更高）
- Python（版本3.7或更高）

## 第二步：构建站点

```bash
npm run mkdocs:build
```

这会生成静态站点到 `site/` 目录。

## 第三步：启动开发服务器

```bash
npm run dev
```

或者双击运行 `start.bat` 文件（Windows系统）

等待几秒钟，你会看到：
- ✅ 后端API服务器运行在 http://localhost:3001
- ✅ MkDocs开发服务器运行在 http://localhost:8000

## 第四步：打开浏览器

访问：**http://localhost:8000** 查看回忆录网站

## 第五步：上传回忆

1. **访问上传页面**
   - 访问：http://localhost:3001/upload
   
2. **填写回忆信息**
   - 标题（必填）
   - 日期（必填，默认为今天）
   - 记录人（可选）
   - 内容（可选）
   - 上传图片（可选，最多10张，单张最大10MB）

3. **保存**
   - 点击"保存回忆"按钮
   - 系统会自动生成Markdown文件并重新构建站点

4. **查看**
   - 刷新 http://localhost:8000 即可看到新回忆

## 📸 关于图片

- 图片保存在 `docs/images/` 目录
- 支持的格式：.jpg, .jpeg, .png, .gif, .webp
- 单张图片最大10MB
- 每次最多上传10张

## 🛠️ 常见问题

### MkDocs命令未找到？

确保已安装Python和pip，然后运行：
```bash
pip install -r requirements.txt
```

### 端口被占用？

如果8000或3001端口被占用：
- 修改 `mkdocs.yml` 中的端口配置
- 修改 `server/index.js` 中的PORT（后端）

### 依赖安装失败？

- 确保已安装 Node.js（版本14或更高）
- 确保已安装 Python（版本3.7或更高）
- 尝试删除 `node_modules` 文件夹后重新安装
- Windows用户可能需要以管理员身份运行

### 上传后站点未更新？

- 检查 `mkdocs build` 是否成功执行
- 查看服务器日志
- 手动运行 `npm run mkdocs:build`

### 图片无法显示？

- 确保图片保存在 `docs/images/` 目录
- 检查Markdown中的图片路径是否正确
- 确保图片文件已提交到Git仓库（如果使用Git）
