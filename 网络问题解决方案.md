# 🌐 Git推送网络问题解决方案

## 🔍 当前问题

无法连接到GitHub（端口443连接失败），这通常是网络或防火墙问题。

## ✅ 解决方案（按优先级）

### 方案1：使用SSH代替HTTPS（最推荐）

SSH通常比HTTPS更稳定，特别是在网络不稳定的情况下。

#### 步骤1：检查SSH密钥

```bash
# 检查是否已有SSH密钥
ls ~/.ssh
```

如果没有看到 `id_ed25519` 或 `id_rsa`，需要生成。

#### 步骤2：生成SSH密钥（如果没有）

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

按Enter使用默认设置（不需要设置密码）。

#### 步骤3：复制公钥

```bash
cat ~/.ssh/id_ed25519.pub
```

复制输出的内容（以 `ssh-ed25519` 开头）。

#### 步骤4：添加到GitHub

1. 访问：https://github.com/settings/keys
2. 点击 **"New SSH key"**
3. Title: `recall-project`
4. Key: 粘贴刚才复制的公钥
5. 点击 **"Add SSH key"**

#### 步骤5：修改远程地址为SSH

```bash
git remote set-url origin git@github.com:xz-06/recall.git
```

#### 步骤6：测试SSH连接

```bash
ssh -T git@github.com
```

应该看到：`Hi xz-06! You've successfully authenticated...`

#### 步骤7：推送

```bash
git push origin main
```

### 方案2：配置Git代理（如果在中国大陆）

如果你在中国大陆，可能需要配置代理：

```bash
# 设置代理（替换为你的代理地址和端口）
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# 推送
git push origin main

# 完成后取消代理
git config --global --unset http.proxy
git config --global --unset https.proxy
```

---

### 方案3：使用GitHub Desktop（最简单）

如果命令行一直失败，使用GitHub Desktop：

1. **下载GitHub Desktop**
   - 访问：https://desktop.github.com
   - 下载并安装

2. **登录并添加仓库**
   - 使用GitHub账号登录
   - File → Add Local Repository
   - 选择项目目录

3. **推送代码**
   - 在GitHub Desktop中点击 "Push origin"

---

### 方案4：稍后重试

网络问题可能是暂时的：

1. **等待几分钟后重试**
   ```bash
   git push origin main
   ```

2. **换个网络环境**
   - 使用手机热点
   - 使用VPN
   - 换个网络
